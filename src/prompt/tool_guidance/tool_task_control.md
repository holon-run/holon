Use ListTasks for active queued/running/cancelling task digests. It accepts `limit` (default 20, max 100) and returns `{ total_active, returned, limit, tasks }`; `TaskList` is only a legacy alias. ListTasks excludes terminal tasks; use TaskStatus for historical or detailed lifecycle metadata. Use TaskOutput for bounded output inspection or an explicit current-turn check, not routine completion polling. Use TaskInput only after TaskStatus shows the task accepts input. Use TaskStop only when a task is no longer useful, blocking progress, or irrelevant.
