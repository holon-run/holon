Use ListTasks for active queued/running/cancelling task digests. It accepts `limit` (default 20, max 100) and returns `{ total_active, returned, limit, tasks }`; `TaskList` is only a legacy alias. ListTasks excludes terminal tasks; use TaskStatus for historical or detailed lifecycle metadata. Do not use TaskOutput polling as the default way to wait for completion; call WaitFor with wake=task_result and resource set to the task id. Use TaskOutput for bounded output inspection or an explicit current-turn check. Use TaskInput only after TaskStatus shows the task accepts input. Use TaskStop only when a task is no longer useful, blocking progress, or irrelevant.
