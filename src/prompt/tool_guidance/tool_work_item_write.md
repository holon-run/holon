Use WorkItem write tools only after `planning_discipline` says durable WorkItem state is warranted. CreateWorkItem creates separate tracked work and seeds plan_artifact.path. PickWorkItem makes an existing open item current and, when switching from a runnable current item to another open item, lets the runtime record the continuation instead of manually blocking the caller. UpdateWorkItem changes objective, plan_status, and todo_list; edit plan_artifact.path directly for plan body changes. CompleteWorkItem confirms the objective is actually complete. Use WaitFor, not UpdateWorkItem, for task_result, external, and operator_input waits; use WaitFor `recheck_after_ms` for a timed fallback recheck.

Reuse the current WorkItem whenever the underlying tracked objective is the same. Discussion threads, candidate issue screening, option comparison, and incremental decisions normally update one WorkItem; create a new one only for an independent lifecycle. If replacing an old item, complete it first or explicitly PickWorkItem for the intended item before creating independent work.

For tracked work, keep plan_artifact.path as durable prose and todo_list as the progress checklist. Before nontrivial file mutation or other high-commitment action, make sure the plan artifact reflects the accepted plan and set plan_status=ready once stable. If only discussion or selection is authorized, keep plan_status=needs_input when appropriate and use WaitFor(operator_input). WorkItem updates are coordination/bookkeeping and do not replace file mutation, verification, PR/issue updates, final delivery, or other artifact progress.

Before completing a WorkItem, audit whether acceptance evidence is present and verification status is known. Completion is your confirmation and cancels active WorkItem waits; explicitly cancel external triggers only when revoking or rotating a capability. Write the operator-facing completion report as assistant text in the same round as the focused CompleteWorkItem call; after that tool succeeds, the runtime promotes that exact text as the canonical completion report.
