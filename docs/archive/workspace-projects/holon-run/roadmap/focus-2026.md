# holon-run 优先级与收敛路线图（2026）

## 当前阶段目标

当前阶段不追求“功能面最广”，而追求：

- 主线更清楚
- probe 机制更清楚
- 仓库边界更清楚
- 一个真实闭环更清楚

## 优先级

### P0

- 明确 `holon` 和 `uxc` 的分工
- 明确哪些仓库属于主线
- 停止无边界扩张实验仓库

### P1

- 做一个最能代表主线的 demo 或使用闭环
- 用时间受限的 probe 验证产品形态和关键分层边界
- 让外部人能快速理解：
  - 为什么不是普通 MCP 工具集合
  - 为什么强调 local-first
  - 为什么要有统一接入层

### P2

- 整理 bridge、host、agentvm 这类辅助仓库的定位
- 让它们服务主线，而不是变成并列产品

### P3

- 再考虑更广泛的生态整合和更多协议接入

## 暂时不做

- 不急着新增太多一次性 skill demo
- 不急着扩张成“大而全 agent 平台”
- 不急着把每条实验线都产品化

## 验收标准

做到下面几件事，说明这条线开始收住了：

- 能用一句话讲清楚 `holon-run` 在做什么
- 能明确说出主仓库和辅助仓库
- 新方向出现时，有一套稳定的取舍标准
- 外部人能在一个 demo 里理解这条线的价值
